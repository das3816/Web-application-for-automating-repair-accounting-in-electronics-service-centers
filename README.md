Опис проєкту
Даний проєкт являє собою веб-застосунок для автоматизації обліку ремонтів у сервісних центрах електроніки. Система дозволяє користувачам створювати заявки на ремонт техніки, спеціалістам виконувати ремонтні роботи та оновлювати інформацію про їх виконання, а адміністраторам контролювати роботу сервісного центру, керувати користувачами, спеціалістами та заявками.
Застосунок реалізований за технологією MERN Stack, яка включає MongoDB, Express.js, React.js та Node.js.

Основні можливості системи
Користувач має можливість зареєструватися, увійти до системи, створювати заявки на ремонт електронної техніки, переглядати свої заявки, відстежувати статус ремонту, переглядати коментарі спеціаліста та вартість виконаних робіт.
Спеціаліст має доступ до призначених йому заявок, може переглядати інформацію про несправність пристрою, змінювати статус ремонту, додавати коментарі, вказувати використані деталі та вартість ремонту.
Адміністратор має доступ до всіх заявок, може призначати спеціалістів, змінювати статуси, редагувати та видаляти заявки, керувати користувачами та спеціалістами, а також переглядати статистику роботи сервісного центру.

Необхідне програмне забезпечення
Перед запуском проєкту необхідно встановити:
Node.js версії 18 або новішої;
MongoDB Community Server або MongoDB Atlas;
Git;
Visual Studio Code (рекомендовано).

Встановлення проєкту
Спочатку необхідно завантажити або розпакувати проєкт на комп'ютер.
Після цього потрібно відкрити термінал та перейти до папки backend:
cd backend
Встановити всі необхідні залежності:
npm install

Далі потрібно створити файл .env у папці backend та додати до нього такі параметри:
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/repair_center_db
JWT_SECRET=repair_secret_key

Якщо використовується MongoDB Atlas, параметр MONGO_URI потрібно замінити на рядок підключення до хмарної бази даних.

Після налаштування необхідно запустити серверну частину проєкту:
npm run dev

У разі успішного запуску в консолі з'являться повідомлення:
MongoDB Connected
Server running on port 5000

Після цього потрібно відкрити новий термінал та перейти до папки frontend:
cd frontend

Встановити залежності клієнтської частини:
npm install

Запустити клієнтський застосунок:
npm run dev

Після запуску Vite у консолі буде відображено адресу:
http://localhost:5173

Необхідно відкрити її у браузері.

Використання системи
Після запуску застосунку відкриється сторінка авторизації. Для роботи із системою необхідно зареєструвати нового користувача та вибрати одну з ролей:

Користувач;
Спеціаліст;
Адміністратор.

Після входу користувач буде автоматично перенаправлений до відповідного особистого кабінету залежно від ролі.
У кабінеті користувача доступне створення нових заявок на ремонт та перегляд власних заявок.
У кабінеті спеціаліста відображаються лише ті заявки, які були призначені адміністратором.
У адміністративній панелі доступне керування користувачами, спеціалістами, заявками та перегляд статистики роботи сервісного центру.
